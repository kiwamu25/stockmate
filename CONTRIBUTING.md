# Contributing Guide

Thanks for contributing to StockMate.

## Before you start

- Search existing issues and pull requests before creating new ones.
- For larger changes, open an issue first to align on scope.

## Local setup

1. Clone the repository.
2. Start backend:
   - `cd backend`
   - `go run ./cmd/server`
3. Start frontend:
   - `cd frontend`
   - `npm install`
   - `npm run dev`

## Development rules

- Keep changes focused and small.
- Update docs when behavior or API changes.
- Keep naming and style consistent with existing code.

## Pull requests

- Use a clear title and include motivation and impact.
- Link related issues (for example: `Closes #123`).
- Include test steps and screenshots for UI changes.
- Make sure builds and tests pass before requesting review.
